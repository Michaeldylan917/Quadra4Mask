# QuadTask（四象限任务管理）- 高保真原型设计

> 这是一个为HarmonyOS原生应用设计的离线优先任务管理工具，基于艾森豪威尔四象限法则。本项目包含完整的产品分析、UI设计和HTML原型实现。

---

## 📋 项目概览

### 应用信息
- **应用名称**: QuadTask（四象限任务）
- **应用类型**: 离线优先的任务管理/时间管理工具
- **目标平台**: HarmonyOS 原生（ArkTS）
- **数据策略**: 完全本地存储（加密可选）

### 核心价值主张
1. **四象限矩阵** - 基于艾森豪威尔法则的任务优先级管理
2. **离线优先** - 无需网络，所有数据本地存储
3. **多维视图** - 列表、日历、时间线、热力图等多种视角
4. **智能建议** - 基于历史行为的本地规则引擎
5. **专注模式** - 番茄钟+自动勿扰功能
6. **数据安全** - 本地备份、导入/导出、版本历史

## 🎯 用户体验分析

### 目标用户
- **知识工作者** - 需要优先级管理的专业人士
- **学生** - 需要时间管理和任务规划的学生
- **项目经理** - 需要任务分解和进度追踪的管理者

### 核心用户需求
1. **快速添加** - 1-2秒内创建任务

3. **进度追踪** - 可视化的完成情况
4. **专注支持** - 减少干扰，提高效率
5. **数据安全** - 本地存储，隐私保护

### 核心交互逻辑
```

├── 多视图切换 → 日历/时间线/统计

```

## 💻 HTML原型实现

### 项目结构
```
prototype/
├── index.html                 # 主入口页面
├── css/
│   ├── tailwind.css          # Tailwind CSS框架
│   ├── global.css            # 全局样式
│   └── components.css        # 组件样式
├── js/
│   ├── app.js                # 应用主逻辑
│   ├── storage.js            # 本地存储管理
│   ├── utils.js              # 工具函数
│   └── mock-data.js          # 模拟数据
├── pages/
│   ├── splash.html           # 启动页
│   ├── privacy.html          # 隐私声明
│   ├── onboarding.html       # 新手引导
│   ├── home.html             # 主视图
│   ├── add-task.html         # 快速添加
│   ├── task-detail.html      # 任务详情
│   ├── calendar.html         # 日历视图
│   ├── timeline.html         # 时间线视图
│   ├── focus.html            # 专注模式
│   ├── stats.html            # 统计分析
│   ├── settings.html         # 设置页
│   └── help.html             # 帮助中心
└── assets/
    ├── icons/                # 图标资源
    └── images/               # 图片资源
```

### 技术栈
- **HTML5** - 语义化标签
- **TailwindCSS** - 原子化CSS框架
- **FontAwesome** - 图标库
- **Vanilla JavaScript** - 交互逻辑
- **LocalStorage API** - 本地数据存储

### 设计特点
1. **iPhone 15 Pro模拟** - 390x844px屏幕尺寸，圆角化设计
2. **iOS状态栏** - 模拟真实iOS状态栏
3. **底部TabBar** - 类似iOS的应用导航
4. **真实图片** - 使用Unsplash等资源的真实图片
5. **响应式交互** - 支持拖拽、滑动、点击等交互
6. **深色模式** - 支持系统深色模式

## 📞 技术支持

### 常见问题
1. **数据丢失** - 检查浏览器是否禁用LocalStorage
2. **界面显示异常** - 清除浏览器缓存后重新加载
3. **交互不响应** - 检查浏览器控制台是否有错误

### 开发建议
- 使用Chrome/Safari浏览器获得最佳体验
- 在移动设备上使用Chrome开发者工具的设备模拟
- 定期备份重要数据
